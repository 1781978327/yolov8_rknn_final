# yolov8-rk3588-cpp-3-15 运行说明

## 编译（本机 RK3588 上）

```bash
cd /home/orangepi/Desktop/yolov8-rk3588-cpp-3-15
mkdir -p build/native && cd build/native
cmake ../..
make -j$(nproc)
make install
```

编译产物在 `install/rknn_yolov8_demo_Linux/`。若已用 `build-linux_RK3588.sh` 或原有方式编过，可直接用该目录下的可执行文件。

## 运行

进入安装目录后执行：

```bash
cd /home/orangepi/Desktop/yolov8-rk3588-cpp-3-15/install/rknn_yolov8_demo_Linux
```

**用法：** `./rknn_yolov8_demo <模型路径> [视频路径] [输出路径]`

### 0. 推 RTSP（给 VLC/前端看）

前提：本机已启动 RTSP 服务器（例如 mediamtx），监听 `8554`。推流地址形如：

- `rtsp://127.0.0.1:8554/cam0`

推流启动示例（直接运行 build 目录的可执行文件，不依赖 sh）：

```bash
export LD_LIBRARY_PATH=/home/orangepi/Desktop/yolov8-rk3588-cpp-3-15/build/lib:$LD_LIBRARY_PATH
/home/orangepi/Desktop/yolov8-rk3588-cpp-3-15/build/rknn_yolov8_demo \
  /home/orangepi/Desktop/yolov8-rk3588-cpp-3-15/model/RK3588/yolov8s.rknn \
  0 \
  rtsp://127.0.0.1:8554/cam0
```

VLC 拉流：

- 本机：`rtsp://127.0.0.1:8554/cam0`
- 局域网：`rtsp://<RK3588_IP>:8554/cam0`

（如果用的是 mediamtx 且开启了 HLS，前端/浏览器也可用：`http://<RK3588_IP>:8888/cam0/index.m3u8`）

### 1. 用项目里的「摔倒视频」跑一遍并保存结果

```bash
./rknn_yolov8_demo ./model/RK3588/yolov8s.rknn ../../摔倒视频.mp4 result.mp4
```

（按 `q` 退出；结果保存在当前目录 `result.mp4`。）

### 2. 只用摄像头（不保存文件）

```bash
./rknn_yolov8_demo ./model/RK3588/yolov8s.rknn 0
```

### 3. 用摄像头并保存到文件

```bash
./rknn_yolov8_demo ./model/RK3588/yolov8s.rknn 0 output.mp4
```

### 4. 用其他视频文件

```bash
./rknn_yolov8_demo ./model/RK3588/yolov8s.rknn /path/to/你的视频.mp4 /path/to/输出.mp4
```

## 说明

- 模型：当前可用 `model/RK3588/yolov8s.rknn`、`yolov8n.rknn`、`yolov8m.rknn`，任选一个作为第一个参数。
- 视频路径为空或为 `0` 时使用摄像头。
- 输出路径不写则只显示窗口、不写文件。
- 已改为用 FFmpeg 后端打开视频文件，支持中文路径（如 `摔倒视频.mp4`）。

定位到了，确实有实质性问题，而且我已经在代码里修完并做了回归。

**根因**
1. RTSP 发包有内存泄漏  
在 `send_packet()` 里每帧分配包内存但没正确释放，长时间会涨内存，最终 OOM/异常退出。  
位置：[rtsp_mpp_sender.cpp](/home/orangepi/Desktop/web/yolov8-rk3588-cpp-3-15/src/rtsp_mpp_sender.cpp:226)

2. `rtsp/stop` 只关标志、不销毁 sender  
会留下“已停止但流端点还在”的假在线状态（你之前看到的 width=0/空壳流就是这一类）。  
位置：[main_http_ctrl.cc](/home/orangepi/Desktop/web/yolov8-rk3588-cpp-3-15/src/main_http_ctrl.cc:1314)

3. 真正销毁后暴露出双重释放（已修）  
`hwdevice/hwframe` 被赋给 codecCtx 时没加引用，销毁阶段多处 unref，导致 `malloc_consolidate(): invalid chunk size`。  
位置：[rtsp_mpp_sender.cpp](/home/orangepi/Desktop/web/yolov8-rk3588-cpp-3-15/src/rtsp_mpp_sender.cpp:108)  
位置：[rtsp_mpp_sender.cpp](/home/orangepi/Desktop/web/yolov8-rk3588-cpp-3-15/src/rtsp_mpp_sender.cpp:359)

**我做的修复**
